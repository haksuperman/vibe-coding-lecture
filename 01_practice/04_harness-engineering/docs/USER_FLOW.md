# 유저 저니 & 유저 플로우

## 1. 한 줄 컨셉
> "CVE 번호 하나만 던지면 → 이게 뭔지 / 얼마나 위험한지 / 뭘 해야 하는지 / 어떻게 학습할지를 한 화면에 정리해주는 도구"

## 2. 유저 저니 (페르소나)
```
페르소나 A — 보안 담당자 김실무
 상황: "CVE-2024-3094 백도어" 뉴스. 우리 시스템 영향은?
 걱정 ─▶ 검색 ─▶ CVE번호 입력 ─▶ 위험도/영향제품/조치 확인 ─▶ 대응
 성공 = "패치해야 할지 5분 안에 판단"

페르소나 B — 보안 학습자 이주니
 상황: 유명 CVE의 원리와 실습 방법이 궁금
 호기심 ─▶ CVE 입력 ─▶ 분석/유사취약점/실습시나리오 확인 ─▶ 학습
 성공 = "원리 이해 + 안전한 실습 시나리오 확보"
```
핵심: 입력은 단순하게(번호/URL/뉴스 아무거나), 출력은 풍부하게(4섹션).

## 3. 유저 플로우 — 화면 4상태 (단일 페이지)
```
 ① 빈 상태 ──[분석 클릭]──▶ ② 로딩 중 ──[완료]──▶ ③ 결과 표시
     ▲                          │                      │
     │ "다시 입력"               │ [실패]                │ "새 분석"
     └──────────────── ④ 에러 상태 ◀──────────────────┘
```

빈 상태 / 결과 상태 와이어프레임:
```
┌─ ① 빈 상태 ───────────────────┐   ┌─ ③ 결과 표시 ────────────────────┐
│  CVE Insight                 │   │  CVE-2024-3094  🔴 CVSS 10.0     │
│ ┌──────────────────────────┐ │   │ ──────────────────────────────── │
│ │ CVE번호·URL·뉴스 붙여넣기 │ │   │ ▸ 📋 취약점 분석/설명             │
│ └──────────────────────────┘ │   │ ▸ 🛠 조치방안          [우선순위] │
│ 예시:[CVE-2024-3094][Log4Shell]│  │ ▸ 🔗 유사/관련 취약점             │
│                  [ 분석하기 ▶]│   │ ▸ 🧪 실습 시나리오 (학습용)       │
└──────────────────────────────┘   │                    [ 새 분석 ↺ ] │
                                    └──────────────────────────────────┘
```

## 4. 시스템 플로우 (입력 → 출력)
```
[브라우저]              [Next.js API 라우트]                [외부]
 사용자 입력 ─POST /api/analyze─▶
                     ① extractCveId  (정규식)
                     ② fetchCveMetadata ──────▶ NVD API
                        (번호 있을 때만, 실패→null)
                     ③ analyzeCve ────────────▶ OpenAI
                        (NVD 사실 + 입력 → 4섹션)
 결과 4섹션 ◀─JSON 응답─ ④ 조합
```
설계 의도:
- 사실은 NVD에서, 해석·시나리오는 OpenAI에서 → LLM 환각(특히 CVSS) 억제
- NVD 실패해도 멈추지 않음 → 입력 텍스트만으로 분석 (폴백)
- API 키는 서버에서만 → 브라우저에 노출 안 함

## 5. 결과 4섹션
| 섹션 | 답하는 질문 | 출처 |
|------|------------|------|
| 📋 분석/설명 | 이게 뭐고 왜 위험해? | NVD 설명 + OpenAI |
| 🛠 조치방안 | 나 뭘 해야 해? | OpenAI (NVD 영향버전 기반) |
| 🔗 유사/관련 취약점 | 비슷한 게 또 있어? | OpenAI |
| 🧪 실습 시나리오 | 어떻게 학습/검증해? | OpenAI |
상단에 항상 CVE 번호 + CVSS 위험도 배지(🔴Critical/🟠High/🟡Medium) 고정.

## 6. MVP 범위
```
✅ 포함: 단일 입력→4섹션, NVD 보강+폴백, 다크 미니멀 단일 페이지
❌ 제외: 이력 저장/DB, 계정/로그인, 자동 모니터링·알림, 리포트 내보내기
```
