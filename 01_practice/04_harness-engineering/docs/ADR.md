# Architecture Decision Records

## 철학
MVP 속도 최우선. 외부 의존성 최소화. 작동하는 최소 구현을 선택한다. 보안(특히 API 키 노출 방지)은 타협하지 않는다.

---

### ADR-001: Next.js 15 App Router + Route Handler를 백엔드로 사용
**결정**: 별도 백엔드 서버 없이 Next.js App Router의 Route Handler(`app/api/analyze`)에서 외부 API를 호출한다.
**이유**: 단일 코드베이스로 프론트+백엔드를 처리해 MVP 속도 확보. 서버 전용 코드에서 API 키를 안전하게 사용 가능.
**트레이드오프**: 무거운 백그라운드 작업/스케줄링에는 부적합하나 MVP 범위에선 불필요.

### ADR-002: CVE 데이터는 NVD 공식 API + OpenAI 조합으로 확보
**결정**: CVE 번호가 있으면 NVD REST API(v2)로 공식 메타데이터를 조회하고, 그 사실 데이터를 OpenAI에 컨텍스트로 주입해 분석·시나리오를 생성한다.
**이유**: LLM 단독은 최신 CVE/정확한 CVSS에 약하다. 사실은 NVD에서, 해석·조치·시나리오는 LLM에서 받아 정확도와 유용성을 모두 확보.
**트레이드오프**: NVD API rate limit(키 없이 분당 제한) 및 외부 장애에 영향받음. NVD 조회 실패 시 입력 텍스트만으로 분석하도록 폴백한다.

### ADR-003: Stateless — 영속 저장소 미도입
**결정**: 분석 결과를 DB/파일에 저장하지 않고 매 요청마다 새로 생성한다.
**이유**: MVP 범위에서 이력 관리는 불필요. 인프라 단순화로 개발·배포 속도 확보.
**트레이드오프**: 동일 CVE 재분석 시 캐시 이점 없음. 추후 캐싱/이력 도입 여지는 남겨둔다.

### ADR-004: 비밀키는 환경변수로만 관리, 클라이언트 노출 금지
**결정**: `OPENAI_API_KEY`, (선택)`NVD_API_KEY`는 `.env.local`에만 두고, 서버 전용 코드에서만 읽는다. `.env.example`로 키 목록을 문서화하고 `.env.local`은 gitignore한다.
**이유**: API 키 유출 방지는 보안의 기본. `NEXT_PUBLIC_` 접두사를 쓰지 않음으로써 클라이언트 번들 유입을 차단.
**트레이드오프**: 없음(필수 보안 조치).
