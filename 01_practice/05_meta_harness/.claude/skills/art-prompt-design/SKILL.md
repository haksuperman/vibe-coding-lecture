# Art Prompt Design — GPT Image 2 작화 프롬프트

콘티 컷을 **OpenAI GPT Image 2**에 입력할 프롬프트로 변환한다. GPT Image 2의 특성에 맞춘 설계가 품질을 좌우한다.

## GPT Image 2 특성 (설계 근거)
- **자연어 서술형에 강하다** — 태그 나열("1girl, blue hair")이 아니라 완결된 문장으로 장면을 묘사할 때 가장 잘 따른다.
- **높은 지시 충실도** — 위치·인원수·관계·동작 같은 구체적 지시를 잘 지킨다. 모호함보다 명시가 낫다.
- **레퍼런스 이미지 입력 지원** — `images.edit`에 기존 이미지를 함께 넣어 동일 캐릭터·스타일을 이어갈 수 있다. 캐릭터 일관성의 1차 수단.
- **이미지 내 텍스트 렌더링 가능** — 단 한글은 깨질 수 있다. 말풍선 대사는 비우고 후편집 권장.

## 선행 조건
- `_workspace/02_storyboard_ep{N}.md`, `_workspace/00_character_sheets.md`가 있어야 한다. 없으면 멈추고 선행 에이전트를 요청한다.

## 산출 구조 → `_workspace/03_art_prompts_ep{N}.md`

```markdown
# {작품명} {N}화 작화 프롬프트 (GPT Image 2)

## 전역 스타일 블록        // 모든 프롬프트 앞에 공통 삽입, 화풍 고정
Style: soft cel-shaded Korean webtoon illustration, clean lineart, ...

## 캐릭터 고정 외형 블록     // 등장 컷마다 그대로 복사해 일관성 유지
### {캐릭터명}
A {나이대} character with {머리·눈·체형·의상·소품}, ...

## 컷 1 프롬프트
- Prompt (EN): "{전역 스타일} {장면·인물·앵글·조명·감정 자연어 서술}. {등장 캐릭터 고정 외형 블록 삽입}."
- Size: 1024x1536 (portrait)   // 세로 웹툰 기본
- Reference: {1번 컷은 신규 / 이후는 직전 동일 인물 컷을 레퍼런스로}
- 주석(KR): {이 컷이 무엇을 노리는지 한국어 설명}
## 컷 2 프롬프트
...
```

## 작성 원칙 (왜)

- **자연어로 쓴다**: 문맥과 지시를 이해하는 모델이므로, 장면을 사람에게 설명하듯 문장으로 묘사한다.
- **일관성 = 반복 + 레퍼런스**: ① 캐릭터마다 고정 외형 블록을 만들어 등장 컷마다 동일 문장을 삽입한다. ② 첫 등장 컷의 결과 이미지를 레퍼런스로 삼아 이후 컷에 `images.edit` 다중 입력으로 넘겨 동일 인물을 유지하라고 명시한다.
- **스타일 전역 고정**: 작품 화풍을 전역 스타일 블록 한 곳에 정의하고 모든 컷 프롬프트 앞에 붙인다. 컷마다 다른 스타일 단어를 쓰지 않는다.
- **텍스트는 비운다**: 한글 대사 렌더링은 불안정하므로 말풍선은 빈 채로 생성하고 후편집한다. 의도된 SFX만 짧은 영문으로 요청.
- **세로 사양 고정**: 웹툰은 세로 매체다. portrait 사이즈(예: 1024×1536)를 기본으로 하고 컷 특성(긴 임팩트 컷 등)에 맞춰 조정한다.
- **영문 프롬프트 + 한국어 주석 병기**: 모델 입력은 영문, 사용자 이해용 주석은 한국어.

## 컷↔콘티 매핑
- 컷 번호는 콘티와 1:1로 유지한다. 콘티에 없는 컷을 만들거나 번호를 바꾸지 않는다.

## 재실행
- 프롬프트 파일이 있으면 읽고 변경 콘티/피드백 컷만 갱신. 전역 스타일·캐릭터 블록은 가급적 유지해 일관성을 지킨다.
