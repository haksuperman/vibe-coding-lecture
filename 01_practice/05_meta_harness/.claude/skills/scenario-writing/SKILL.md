# Scenario Writing — 웹툰 에피소드 집필

세계관·캐릭터 바이블을 바탕으로 **한 화 분량의 대본**을 쓴다. 산출물은 콘티 감독이 컷으로 분해할 수 있는 씬 단위 시나리오다.

## 선행 조건
- `_workspace/00_world_bible.md`, `_workspace/00_character_sheets.md`가 있어야 한다. 없으면 멈추고 `story-architect` 선행을 요청한다.

## 산출 절차

1. **이전 화 확인** — 연속 화면 이전 화의 훅·떡밥을 이어받는다.
2. **한 화 감정 곡선 설계** — 도입 → 전개 → 절정 → 훅(다음 화 유인). 평탄하게 끝내지 않는다.
3. **씬 단위 집필** → `_workspace/01_scenario_ep{N}.md`

## 시나리오 구조

```markdown
# {작품명} {N}화 — {부제}
## 한 줄 요약
## 씬 1
- 장소: {배경}
- 등장인물: {이름들}
- 지문: {감정·동작·배경 변화 서술}
- 대사:
  - {캐릭터}: "{대사}"
## 씬 2
...
## 다음 화 훅
```

## 작성 원칙 (왜)

- **캐릭터 보이스 고정**: 캐릭터 시트의 말투(어미·1인칭·말버릇)를 어기지 않는다. 보이스가 흔들리면 독자가 인물을 못 알아본다.
- **하나의 감정 곡선 + 끝 훅**: 웹툰 한 화는 하나의 감정 절정과, 다음을 보게 만드는 마지막 컷을 가진다. 훅 없는 화는 이탈을 부른다.
- **짧은 입말 대사**: 세로 스크롤에서 긴 대사 덩어리는 가독성을 해친다. 짧게, 입말로.
- **보여줄 것 / 말할 것 분리**: 시각으로 전달 가능한 정보는 대사로 설명하지 말고 지문으로 넘겨 콘티 감독이 연출하게 한다("show, don't tell").
- **신규 설정 금지**: 새 캐릭터·설정이 필요하면 임의로 만들지 말고 `story-architect`에게 요청해 바이블에 먼저 반영한다.

## 재실행
- 해당 화 파일이 있으면 읽고 피드백 부분만 수정. 시나리오 변경은 콘티·작화에 연쇄되므로 변경 범위를 명확히 표시한다.
