# Consistency Review — 웹툰 일관성 검수

웹툰 파이프라인의 품질 게이트. 핵심은 "존재 확인"이 아니라 **경계면 교차 비교** — 인접한 두 문서를 동시에 읽고 shape이 어긋나는 지점을 찾는다.

## 검수 축과 경계면

| 축 | 비교 대상(경계면) |
|----|------------------|
| 캐릭터 일관성 | 캐릭터 시트 ↔ 시나리오 대사 ↔ 콘티 등장인물 ↔ 작화 고정 외형 블록 |
| 플롯 일관성 | 세계관 규칙 ↔ 시나리오 전개(규칙 위반·떡밥 미회수·시간선 모순·화차 연속성) |
| 톤 일관성 | 바이블 무드 ↔ 시나리오 ↔ 연출 ↔ 전역 스타일 블록 |
| 연출/매체 | 세로 스크롤 호흡, 컷 번호 매핑(콘티 ↔ 프롬프트 1:1) |

## 검수 절차 (점진적)

각 단계 산출물이 나올 때마다 **즉시 해당 경계면만** 검수한다. 전체 완성 후 1회가 아니다 — 늦게 잡힌 모순일수록 재작업 비용이 크다.

1. 직전 생성된 파일과 그 **인접 상류 파일**을 나란히 읽는다.
2. 위 표의 해당 축을 교차 비교한다.
3. 발견을 심각도·위치·수정 대상으로 기록 → `_workspace/qa_report_ep{N}.md`

## 리포트 구조

```markdown
# QA 리포트 — {N}화 (검수 시점: {단계})
## Blocker     // 다음 단계 진행을 막는 모순
- [위치] {파일·컷번호}: {무엇 ↔ 무엇 충돌} → 수정 대상: {에이전트} / 제안: {…}
## Major
- ...
## Minor
- ...
## 회귀 확인    // 이전 리포트 지적이 반영됐는지
- ...
```

## 원칙 (왜)

- **교차 비교 우선**: 한 문서만 읽으면 내부 정합만 본다. 일관성 버그는 문서 사이 경계에서 터진다 — 반드시 두 문서를 나란히 본다.
- **삭제 금지, 출처 병기**: 상충 데이터는 지우지 말고 "A파일의 X ↔ B파일의 Y 충돌"로 출처를 함께 적는다.
- **직접 수정 금지**: 검수자는 고치지 않는다. 수정은 책임 전문 에이전트의 몫이고, 검수자는 재검수로 닫는다.
- **blocker 즉시 통지**: blocker는 리포트에만 남기지 말고 책임 에이전트·리더에 즉시 알려 진행을 멈춘다.

## 재실행
- 이전 QA 리포트가 있으면 읽고 회귀 확인 후 신규 이슈만 추가한다.
