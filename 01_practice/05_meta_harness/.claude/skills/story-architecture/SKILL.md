# Story Architecture — 웹툰 기획 설계

웹툰의 토대를 만든다. 여기서 만든 세계관·캐릭터는 시나리오·콘티·작화가 평생 참조하는 단일 진실 공급원이다. 따라서 **시각적으로 구체적이고 모순 없는** 설계가 핵심이다.

## 산출 절차

0. **제작 기획안 먼저 작성 (승인 게이트용)** → `_workspace/00_plan.md`
   - 상세 바이블·캐릭터 시트를 만들기 **전에**, 방향을 합의할 간단한 기획안을 먼저 낸다. 잘못된 방향으로 전체를 생성하는 낭비를 막기 위함이다.
   - 포함: 로그라인(한 문장), 장르·톤·무드, 타깃 독자, 주요 캐릭터 3~5명 한 줄 소개, 전체 연재 구조 + 이번 화 구성(도입→절정→훅) 한 단락.
   - 오케스트레이터가 이 기획안으로 사용자 승인을 받는다. **승인 전에는 아래 1~4단계(상세 생성)로 넘어가지 않는다.** 수정 요청이 오면 기획안만 고쳐 재제시한다.
1. **장르·소재·톤 확정** — 승인된 기획안을 기준으로 한다. (기획안 단계에서 불명확하면 1개 핵심 질문으로 좁힌다.)
2. **세계관 바이블 작성** → `_workspace/00_world_bible.md` (기획안을 구체화)
3. **캐릭터 시트 작성** → `_workspace/00_character_sheets.md` (기획안 캐릭터를 시각적으로 구체화)
4. **전체 플롯 아크 작성** → 바이블 하단 또는 별도 섹션

## 세계관 바이블 구조

```markdown
# 세계관 바이블: {작품명}
## 로그라인        // 한 문장 요약
## 장르·톤·무드     // 시각 톤 포함 (밝은/어두운, 채도)
## 배경            // 시대·공간·사회 체계
## 핵심 규칙        // 마법/기술/제약. 모순 없이 닫힌 체계
## 핵심 갈등        // 작품을 끌고 가는 동력
## 플롯 아크        // 기승전결 또는 연재 구조, 떡밥/회수 지점
```

## 캐릭터 시트 구조 (작화 일관성의 원천)

캐릭터 외형은 **작화 프롬프트가 그대로 재사용**한다. 추상 형용사가 아니라 시각적 명사로 적는다.

```markdown
## {캐릭터명}
- 외형: 머리(색·길이·스타일), 눈(색·인상), 체형·키, 기본 의상, 상징 소품, 식별 포인트
- 성격: 핵심 기질 + 결함
- 말투: 어미·1인칭·존댓말 여부·말버릇 (시나리오 작가가 보이스로 사용)
- 동기: 원하는 것 / 두려워하는 것
- 관계: 다른 캐릭터와의 관계
```

## 작성 원칙 (왜)

- **시각적 구체성**: "멋진 기사"가 아니라 "은발 단발, 왼눈 흉터, 검은 망토에 은빛 늑대 문장". 작화 일관성은 이 구체성에서 나온다.
- **닫힌 규칙 체계**: 세계관 규칙이 뒤 화에서 깨지면 전체가 무너진다. 예외를 만들 거면 예외의 비용도 함께 정의한다.
- **세로 스크롤 전제**: 웹툰은 짧은 호흡 매체다. 말로 설명할 설정보다 **컷으로 보여줄 수 있는 갈등**을 우선 설계한다.
