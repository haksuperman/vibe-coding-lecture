# Storyboard Direction — 세로 스크롤 콘티 연출

시나리오를 **세로 스크롤 웹툰의 컷 단위 콘티**로 분해한다. 웹툰은 영화도 출판만화도 아닌 고유 문법(세로 흐름·스크롤 호흡·여백 연출)을 가진다 — 이를 항상 전제한다.

## 선행 조건
- `_workspace/01_scenario_ep{N}.md`가 있어야 한다. 없으면 멈추고 `scenario-writer` 선행을 요청한다.

## 산출 절차

1. **씬→컷 분해** — 각 씬을 정보 전달 순서에 따라 컷으로 나눈다.
2. **컷별 연출 지정** — 샷 사이즈·앵글·구도·캐릭터·말풍선·SFX.
3. **세로 리듬 점검** — 스크롤 호흡, 임팩트 컷 배치, 여백 활용.
4. → `_workspace/02_storyboard_ep{N}.md`

## 콘티 구조

```markdown
# {작품명} {N}화 콘티
## 컷 1
- 등장인물: {이름들}    // 일관성 추적용, 누가 나오는지 명시
- 샷·앵글: {클로즈업/바스트/풀샷/부감/앙각 …}
- 구도 설명: {화면에 무엇이 어떻게 배치되는가 — 시각적으로}
- 대사·SFX: {말풍선 내용 / 효과음}
- 연출 노트: {여백·스크롤 속도·시선 유도}
## 컷 2
...
```

## 연출 원칙 (왜)

- **스크롤 리듬이 핵심**: 정보는 위→아래 순서로 읽힌다. 컷 크기와 컷 사이 여백으로 호흡(빠른 대화 vs 정적·긴장)을 통제한다.
- **임팩트 컷은 크고 단독으로**: 절정·반전은 큰 세로 컷으로 단독 배치. 평범한 대화는 작은 컷으로 빠르게 흘린다.
- **작화로 바로 변환 가능하게**: 각 컷은 작화 프롬프트 디자이너가 즉시 이미지로 옮길 만큼 시각적이어야 한다 — 누가/어디서/무엇을/어떤 앵글로.
- **컷 번호는 영구 키**: 컷 번호는 작화 프롬프트·QA가 동일하게 참조한다. 한번 부여한 번호 체계를 유지한다.
- **외형은 캐릭터 시트 준수**: 임의 외형 변경 금지.

## 재실행
- 콘티 파일이 있으면 읽고 피드백/변경된 시나리오 영향 컷만 갱신. 컷 번호 체계는 유지한다.
