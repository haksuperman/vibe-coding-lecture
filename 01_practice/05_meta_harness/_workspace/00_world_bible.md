# 세계관 바이블: 커밋 가드 — 클로디

## 로그라인
야근하던 주니어 개발자가 API 키를 코드에 하드코딩한 채 푸시하려는 순간, 의인화된 AI 짝꿍 '클로디'가 키 유출 직전에 막아준다.

## 장르·톤·무드
- 장르: 개발자 일상 · 보안 계몽 코미디 (의인화 마스코트물).
- 톤: 밝고 따뜻한 코지 코미디. 위협은 귀엽고, 교훈은 분명하며, 결말은 안도감.
- 시각 톤: 늦은 밤 1인 작업실. 따뜻한 스탠드 주황빛 ↔ 모니터 청색광의 대비가 기본 무드. 부드러운 셀셰이딩, 깔끔한 라인아트, 채도 중상.
- 존재 구분 규칙: **현실 인물(도현)은 일반 채색**, **디지털 존재(클로디·키도깨비)는 은은한 글로우/글리치**로 그려 한눈에 구분한다.

## 배경
- 시대·공간: 현대. 개발자 도현의 늦은 밤 1인 작업실(책상, 듀얼 모니터, 키보드, 별 무늬 머그컵).
- 사회 체계: 평범한 IT 실무 현장. 코드는 git으로 버전관리되며, public 레포에 푸시하면 외부에 공개된다는 현실 규칙이 그대로 작동한다.

## 핵심 규칙 (모순 없는 닫힌 체계)
1. **의인화 존재는 디지털 영역에서 깨어난다.** 클로디는 도현의 터미널/Claude Code 안에 상주하고, 키도깨비는 '코드에 하드코딩된 시크릿'이 푸시 위협을 받을 때 깨어난다. 즉 위협 행동(커밋/푸시)이 트리거다.
2. **클로디의 권능은 "사전 차단(prevention)"으로 한정된다.** 이미 유출된 것을 되돌리는 마법은 없다. 그래서 항상 *푸시 직전*에 개입한다 — 이 제약이 작품의 긴장과 교훈의 핵심.
3. **키도깨비는 공개(public)로 나가야만 진짜 위협이 된다.** `.env` 분리 + `.gitignore` 등록으로 '봉인'되면 무력화된다. 봉인은 삭제가 아니라 격리다(로컬엔 남고 추적만 제외).
4. 도현은 클로디·키도깨비를 볼 수 있다(작가적 약속). 단 이는 도현의 작업 화면 안에서 일어나는 시각화이며, 현실 물리를 깨지 않는다.

## 핵심 갈등
'빨리 동작시키고 싶은 개발자의 조급함' vs '안전하게 지키려는 클로디'. 매 화 새로운 보안 실수가 키도깨비류 위협으로 의인화되어 등장하고, 클로디가 사전에 막아낸다.

## 플롯 아크 (옴니버스 연재)
- 1화: 하드코딩된 시크릿/API 키 (← 이번 화)
- 2화: .env 유출 · .gitignore 누락
- 3화: SQL 인젝션 (1화 훅에서 예고)
- 4화: 의존성 취약점
- 회수 지점: 1화 마지막 컷에서 'SQL 인젝션 그림자'를 떡밥으로 심고 3화에서 회수. 매 화 '실수→의인화 위협→사전 차단→올바른 습관' 포맷을 반복해 계몽 메시지를 누적한다.
