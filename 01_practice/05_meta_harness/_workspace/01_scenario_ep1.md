# 커밋 가드 — 클로디 1화 — 푸시 직전

## 한 줄 요약
하드코딩된 API 키를 무심코 푸시하려는 도현을, 키 유출 직전에 클로디가 막아내고 안전하게 봉인한다.

## 씬 1 — 무심한 커밋 (도입)
- 장소: 깊은 밤 도현의 1인 작업실. 듀얼 모니터에 코드 에디터와 터미널.
- 등장인물: 도현
- 지문: 도현이 하품하며 별 무늬 머그컵을 내려놓는다. 코드 한 줄 `API_KEY = "sk-live-..."`가 화면에 또렷이 보인다. 보안 따위 생각할 겨를 없이, 빨리 자고 싶은 표정. 터미널에 `git commit -m "기능 완성" && git push`를 타이핑하고 엔터 키 위에 손가락을 올린다. (※ 위험은 대사로 설명하지 말고 화면의 코드로 보여줄 것.)
- 대사:
  - 도현: "휴… 드디어 됐다. 바로 올리고 자야지."

## 씬 2 — 잠깐, 그거 진짜 키예요! (절정·보안사고 포착)
- 장소: 동일 작업실, 화면 클로즈업에서 모니터 속 공간으로 확장.
- 등장인물: 도현, 키도깨비, 클로디
- 지문: 엔터를 누르는 순간 코드 줄 사이에서 키도깨비가 'sk-live-...' 라벨을 흔들며 튀어나와 화면 밖 'public repo'라 적힌 빛의 문 쪽으로 도망친다. 동시에 터미널에서 코랄빛 클로디가 솟아올라 키도깨비의 멱살을 붙잡아 푸시를 가로막는다. 터미널엔 빨간 경고 `Blocked: hardcoded secret detected (sk-live-...)`가 떠 있다. 도현의 졸음이 확 달아나 눈이 커진다.
- 대사:
  - 클로디: "잠깐, 그거 진짜 키예요!"
  - 키도깨비: "히히, 밖으로 나갈 거지롱~"
  - 도현: "헉… 그거 그대로 올라갈 뻔했네?!"

## 씬 3 — 봉인 그리고 다음 위협 (훅)
- 장소: 동일 작업실.
- 등장인물: 도현, 클로디, (그림자) 다음 위협
- 지문: 클로디가 키를 `.env` 파일 상자에 넣고 `.gitignore`에 한 줄을 추가하자, 키도깨비가 자물쇠 채워진 상자 안에서 버둥대다 잠잠해진다. 터미널엔 초록색 `Safe: key moved to .env, added to .gitignore`. 도현이 식은땀을 닦으며 안도한다. 클로디가 한쪽 눈(점)을 찡긋한다. 화면 구석 어둠 속에서 'SQL 인젝션'이라 적힌 또 다른 그림자 눈동자가 번뜩인다. (※ 다음 화 떡밥 — 그림자는 대사로 설명하지 말고 실루엣으로만.)
- 대사:
  - 클로디: "키는 .env로 옮기고 .gitignore에 넣었어요. 이제 안전해요."
  - 도현: "고마워 클로디… 너 없었으면 큰일 날 뻔했다."
  - 클로디: "그럼… 다음 녀석도 막아볼까요?"
